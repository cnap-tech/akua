          Why might this fail?
          

            The playground pulls charts directly from the browser, which
            means the Helm repository must send a permissive
            Access-Control-Allow-Origin header. Many OCI
            registries (including ghcr.io) don't — that's a registry
            limitation, not an akua one. Use the
            CLI or the Node SDK
            for those.
          

        

      


      

        

          What exactly is running here?
          

            @akua/sdk/browser (shipped on
            JSR). Same Rust core
            the akua CLI uses, compiled to WebAssembly via
            wasm-pack. The OCI + HTTP Helm pull pipeline
            (bearer token dance, sha256 digest verification, SSRF-safe
            host validation, size caps) is pure-JS fetch on top of the
            WASM manifest-parse / tar-unpack / schema-merge layer.
          

          

            Nothing is proxied. Your browser is talking directly to the
            Helm repo server. You can verify in DevTools Network tab.
          

        


        

          Why not arbitrary OCI registries?
          

            The HTTP pipeline works against anything that sends CORS
            headers. Known-good: Jetstack, Grafana, JFrog (public
            repos). oci://ghcr.io/… and
            oci://docker.io/… don't — both registries omit
            Access-Control-Allow-Origin, and the browser
            will refuse. The Rust CLI and Node SDK have no such
            constraint.
