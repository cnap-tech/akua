# @akua/sdk — spike scaffold

Pre-alpha. One verb wired end-to-end (`akua.version()`) to prove the
pattern. Every other method listed in [`docs/sdk.md`](../../docs/sdk.md)
follows the same shape: spawn `akua <verb> --json`, parse stdout,
return a value whose TypeScript type is auto-generated from the Rust
serde type that defines the `--json` output.

## Types are derived, not hand-written

`src/types/*.ts` are generated by `ts-rs` from the serde types in
`akua-core` and `akua-cli`. `src/schemas/akua.json` is the same shape
as one bundled JSON Schema document, written by `schemars` — polyglot
consumers (Python, Go, another Rust, agents) can pull that file and
validate against it.

Repo tasks:

```sh
task sdk:gen             # regenerate src/types + src/schemas from Rust
task sdk:check           # regenerate then fail on diff (drift guard — also in `task ci`)
task sdk:test            # bun test packages/sdk
task sdk:publish:check   # dry-run `jsr publish` — catches manifest issues before tag push
```

## Release flow

1. Land changes on `main`, ensure `task ci` is green (includes
   `sdk:check` drift guard).
2. Bump `jsr.json` `version` mentally (it's overwritten from the tag
   at publish time — no commit needed).
3. Run `task sdk:publish:check` locally to confirm the manifest +
   slow-type check pass.
4. Push a tag `sdk-v<semver>`:

   ```sh
   git tag sdk-v0.1.0
   git push origin sdk-v0.1.0
   ```

5. `.github/workflows/sdk-release.yml` regenerates types + schema on
   the runner (so a stale local checkout can't leak), runs the drift
   guard, and publishes to JSR via OIDC (no PAT required).

SDK versions float independently of the Rust crate version. A wrapper-
layer fix ships without rebuilding the binary.

## Usage

```ts
import { Akua } from '@akua/sdk';

const akua = new Akua();
const v = await akua.version();
console.log(v.version);  // typed as VersionOutput.version: string
```

## Not in this spike

- Runtime JSON Schema validation at the parse boundary (ajv / Zod
  derived from `akua.json`). Today the SDK trusts the CLI; in
  production we'd validate and throw a typed error on contract drift.
- Remaining ~29 verbs from `docs/sdk.md`.
- `@akua/sdk/browser` entry point (the read-only subset that runs via
  WASM instead of shell-out). Needs an `akua-wasm` crate; see
  `docs/sdk.md` §"Two entry points".
- Publish pipeline (JSR or npm), `.d.ts` generation, type-sync CI step.
