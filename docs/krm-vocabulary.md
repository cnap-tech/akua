# KRM vocabulary

The opinionated shared language of akua. Every platform concern is expressed as a typed resource; the exact shape depends on whether the resource reaches a Kubernetes cluster or lives purely in the akua control plane.

---

## Two classes of resources

Earlier drafts of this spec treated every akua-defined thing as a KRM resource (with `apiVersion` / `kind` / `metadata` envelope). That was over-applied вҖ” cargo-culting Kubernetes shape onto things that never reach a cluster. The corrected split:

**Cluster-facing resources** reach a Kubernetes cluster (as CRs or as generated manifests). Full KRM envelope. Canonical form is KCL; YAML is an equivalent serialized view.

**Control-plane resources** live purely within the akua workflow вҖ” in git, in CI, in the review surface. Typed KCL documents without a Kubernetes envelope. Optional YAML view for interchange.

| resource | class | canonical | YAML view? |
|---|---|---|---|
| `App` | cluster-facing | KCL | вң“ |
| `Environment` | cluster-facing | KCL | вң“ |
| `Cluster` | cluster-facing | KCL | вң“ |
| `Secret` / `SecretStore` | cluster-facing | KCL | вң“ |
| `Gateway` | cluster-facing | KCL | вң“ |
| `Policy` | control-plane | Rego (see [policy-format.md](policy-format.md)) | N/A |
| `Package` | control-plane | KCL (see [package-format.md](package-format.md)) | N/A |
| `Rollout` | control-plane | KCL | вң“ |
| `Runbook` | control-plane | KCL | вң“ |
| `Budget` | control-plane | KCL | вң“ |
| `Incident` | control-plane | KCL | вң“ |
| `Experiment` | control-plane | KCL | вң“ |
| `Tenant` | control-plane | KCL | вң“ |

All canonical forms are typed. The YAML view is generated by `akua export <kind> <name> --format=yaml` and accepted as input by `akua apply -f <file>.yaml`. Agents and the CLI operate on KCL; YAML is for interchange (Slack paste, issue tracker, pipelines that don't embed KCL).

## Cluster-facing resources

### `App`

A per-install instance. References a Package by OCI ref and supplies inputs.

```python
# apps/checkout/app.k
import akua.v1.App
import akua.v1.SecretRef

app: App = {
    name:      "checkout"
    namespace: "payments"
    labels: {
        team: "payments"
        tier: "production"
    }
    spec: {
        package: "oci://pkg.akua.dev/payments-api:3.2"
        inputs: {
            appName:  "checkout"
            hostname: "checkout.example.com"
            replicas: 5
            database: { user: "checkout_app" }
        }
        policy: "tier/production"
        env:    "production"
        secretRefs: [
            SecretRef { name: "stripe-api-key", scope: "read" }
        ]
    }
}
```

Equivalent YAML view (via `akua export`):

```yaml
apiVersion: akua.dev/v1alpha1
kind: App
metadata:
  name: checkout
  namespace: payments
  labels:
    team: payments
    tier: production
spec:
  package: oci://pkg.akua.dev/payments-api:3.2
  inputs:
    appName: checkout
    hostname: checkout.example.com
    replicas: 5
    database:
      user: checkout_app
  policy: tier/production
  env: production
  secretRefs:
    - name: stripe-api-key
      scope: read
```

Fields:

| field | required | notes |
|---|---|---|
| `name` | yes | unique within namespace |
| `namespace` | yes | Kubernetes namespace |
| `labels` / `annotations` | no | standard Kubernetes meta |
| `spec.package` | yes | OCI ref of a Package |
| `spec.inputs` | yes | values matching the Package's `Input` schema |
| `spec.policy` | no | tier name; inherits from Environment if omitted |
| `spec.env` | yes | references an Environment by name |
| `spec.secretRefs` | no | explicit secret grants (Environment can grant broader defaults) |

`App.status` is populated by the reconciler (ArgoCD/Flux/kro). Not authored by humans; read via `akua app get`.

### `Environment`

Per-environment scope: target cluster, policy tier, budget, approvers, soak/rollout defaults.

```python
# environments/production.k
import akua.v1.Environment
import akua.v1.Budget
import akua.v1.SoakRule

env: Environment = {
    name: "production"
    spec: {
        cluster: "prod-eu"
        region:  "eu-central-1"
        policy:  "tier/production"
        budget: Budget {
            monthly:  "$1500"
            alertAt:  [0.75, 0.9, 1.0]
        }
        approvers: ["@team/platform", "@team/payments-lead"]
        soak: [
            SoakRule {
                after_deploy: "15m"
                fail_on:      "error_rate p99 > 0.01 for 3m"
            },
            SoakRule {
                after_deploy: "15m"
                fail_on:      "p99_latency > 2s for 5m"
            },
        ]
    }
}
```

### `Cluster`

Kubernetes cluster metadata + capacity. Usually imported from the platform team's cluster inventory.

```python
cluster: Cluster = {
    name: "prod-eu"
    spec: {
        provider:  "aws-eks"
        region:    "eu-central-1"
        version:   "1.31"
        capacity:  { cpu: "1024 cores", memory: "4Ti" }
        endpoint:  "https://eks-prod-eu.acme.internal"
    }
}
```

### `Secret` / `SecretStore`

Typed secret reference. Values never appear in akua-managed files вҖ” only refs to external secret stores (Vault, Infisical, SOPS).

```python
# secrets/stripe-api-key.k
secret: Secret = {
    name: "stripe-api-key"
    spec: {
        ref:    "vault://secrets/stripe/api-key"
        store:  "acme-vault"
        rotation: { policy: "30d", alertBefore: "7d" }
    }
}

# secret stores
store: SecretStore = {
    name: "acme-vault"
    spec: {
        kind: "vault"
        endpoint: "https://vault.acme.internal"
        authMethod: "kubernetes"
    }
}
```

### `Gateway`

Ingress/egress routing configuration.

```python
gateway: Gateway = {
    name: "public-api"
    spec: {
        hostname: "api.example.com"
        tls:      { issuer: "letsencrypt-prod" }
        routes: [
            { path: "/v1/*",  service: "api",     port: 8080 },
            { path: "/admin", service: "admin",   port: 8080 },
        ]
    }
}
```

## Control-plane resources

### `Package`

See [package-format.md](package-format.md). KCL source + `akua.mod` + `akua.sum`; published as signed OCI artifact. Never has a YAML form; Packages are programs, not documents.

### `Policy`

See [policy-format.md](policy-format.md). Rego source; published as signed OCI artifact. Tiers are Rego packages; customer policies import tier packages and add rules. No YAML form.

### `Rollout`

Cross-repo / cross-service staged deployment orchestration. Can encode conditionals, batch strategies, soak rules, rollback triggers вҖ” use KCL.

```python
rollout: Rollout = {
    name: "upgrade-auth-lib"
    spec: {
        changes: [
            {
                repo:  "github.com/acme/api"
                path:  "apps/api/inputs.yaml"
                patch: { "authLibVersion": "3.2.0" }
            },
            {
                repo:  "github.com/acme/worker"
                path:  "apps/worker/inputs.yaml"
                patch: { "authLibVersion": "3.2.0" }
            },
        ]
        strategy:   "staged"
        batchSize:  10
        soak:       "1h"
        policyTier: "tier/production"
        rollbackOn: [
            "error_rate p99 > 0.02 for 5m",
            "p99_latency > 3s for 5m",
        ]
    }
}
```

### `Runbook`

Agent-executable response playbook. Triggered by observability events; executes typed steps with conditionals and approval gates.

```python
runbook: Runbook = {
    name: "high-error-rate-response"
    spec: {
        trigger: {
            query:    "error_rate p99 > 0.01 for 5m"
            severity: "warn"
        }
        steps: [
            { run: "akua deploy history service=${params.service} --last=3" },
            { run: "akua audit explain change=${last_deploy.change_id}" },
            {
                if: "last_deploy.within == '10m'"
                run: "akua deploy rollback --change=${last_deploy.change_id} --dry-run"
            },
            {
                approval: true
                if:       "dry_run.impact.contains('data')"
            },
            { run: "akua deploy rollback --change=${last_deploy.change_id}" },
            { run: "akua incident create --trigger=${trigger} --action=rollback" },
        ]
    }
}
```

### `Budget`

Cost cap and alert configuration. Referenced by Environment.

```python
budget: Budget = {
    name: "payments-team"
    spec: {
        monthly:     "$1500"
        alertAt:     [0.75, 0.9, 1.0]
        hardCap:     True
        approvers:   ["@team/payments-lead"]
        scope: {
            environments: ["staging", "production"]
            teams:        ["payments"]
        }
    }
}
```

### `Incident`

Structured incident record. Machine-authored by akua's audit spine or learning-loop controller; humans read via `akua audit explain <id>`.

```python
incident: Incident = {
    id: "i-47"
    spec: {
        trigger: {
            type:    "error_rate_spike"
            service: "checkout"
            at:      "2026-04-20T14:08:00Z"
        }
        rootCause: {
            changeId:    "c-4f2c9a"
            actor:       "agent-experiments-4"
            reason:      "enabled new flag X"
            committedAt: "2026-04-20T14:03:00Z"
        }
        resolution: {
            action:      "rollback"
            changeId:    "c-9b3"
            actor:       "agent-incident-responder"
            completedAt: "2026-04-20T14:10:00Z"
        }
        durationMinutes: 7
        learned: "experiment should gate on p99 budget; see policy-template/experiment-v2"
    }
}
```

### `Experiment`

Feature flag / A/B test / canary definition.

```python
experiment: Experiment = {
    name: "new-checkout-flow"
    spec: {
        flag:        "checkout.use_new_flow"
        rolloutPct:  10
        targeting: {
            tier: "production"
            regions: ["eu-central-1"]
        }
        successCriteria: [
            "error_rate p99 < 0.005 for 1h",
            "conversion_rate increase >= 0.5%"
        ]
        abortOn: [
            "error_rate p99 > 0.01 for 3m",
        ]
    }
}
```

### `Tenant`

Multi-tenant isolation unit вҖ” primarily for merchant mode (ISVs shipping to end-customers).

```python
tenant: Tenant = {
    name: "customer-a"
    spec: {
        merchant:  "acme-saas"
        installs:  ["checkout", "inventory", "analytics"]
        contacts:  ["admin@customer-a.example.com"]
        compliance: ["soc2"]
        isolation: {
            namespace: "customer-a"
            network:   "isolated"
        }
    }
}
```

## Canonical form vs YAML view

For any cluster-facing resource (`App`, `Environment`, `Cluster`, `Secret`, `SecretStore`, `Gateway`):

- **Canonical = KCL file.** Authors and agents write KCL. Full type checking, imports, derivation, schema inheritance.
- **YAML view** is deterministically generated: `akua export app checkout --format=yaml > app.yaml`. YAML view matches the KCL one-for-one; `akua apply -f app.yaml` parses the YAML back to a typed in-memory form identical to loading the KCL.

Control-plane resources may have YAML views for interchange (e.g., pasting a Runbook spec in a Slack thread), but the canonical form is always the typed language.

> **`akua export` is not `akua render`.** Export converts a KRM resource into a YAML view of the resource itself вҖ” one file, the App definition. Render evaluates a Package (the program) against inputs and produces deploy-ready Kubernetes manifests вҖ” many files, the actual workload resources. Different verbs, different outputs. See [cli.md вҖ” disambiguation table](cli.md#verb-index).

## Group and version stability

- All kinds use group `akua.dev/v1alpha1` during pre-v1.0 development.
- After v1.0, kinds that prove stable move to `akua.dev/v1`. Unstable kinds remain under `v1alpha1` / `v1beta1`.
- Groupings by functional area (`workload.akua.dev`, `policy.akua.dev`) may come later; for v1alpha1, everything is under `akua.dev`.
