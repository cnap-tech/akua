## How the pieces connect

**Package** (`package/package.k`) вЂ” the typed definition. Would normally live in its own repo and be published as an OCI artifact. Shown inline here for didactic completeness.

**App** (`apps/checkout/app.yaml`) вЂ” references the Package by OCI ref. Contains per-environment inputs. One App resource; renders N times, one per environment.

**Environment** (`environments/{dev,staging,production}.yaml`) вЂ” per-env scope. Carries the policy tier, budget caps, approval rules.

## Render

```sh
# Render for all environments
akua render --all-envs --out ./deploy

# Or for a specific environment
akua render --env production --out ./deploy/production
```

Each render pulls the Package's inputs for that environment, applies the environment's policy tier, and produces committable raw YAML.

## Flow for a change

1. Agent or human edits `apps/checkout/inputs/production.yaml` (e.g. bump replicas).
2. CI runs `akua lint && akua render --env production`.
3. `akua policy check --tier tier/production` вЂ” returns allow/deny/needs-approval.
4. If `needs-approval`: review bot notifies; human approves in browser.
5. PR merges; deploy repo gets updated YAML; Argo/Flux reconciles.
6. `App.status` reflects the reconciled state; visible via `akua app get checkout`.

See the files in this directory for the concrete shapes.
